# todo_manager プロジェクトルール

## バージョン管理ルール

コードを変更するたびに、**必ず以下を更新すること**：

1. **`package.json`** — `version` フィールドをセマンティックバージョニングに従って更新
2. **`CHANGELOG.md`** — 変更内容を該当バージョンのセクションに追記

### バージョン番号の運用

| 変更種別 | 上げるバージョン | 例 |
|----------|------------------|----|
| 機能追加 | マイナー（x.**Y**.z） | 1.3.0 → 1.4.0 |
| バグ修正 | パッチ（x.y.**Z**） | 1.4.0 → 1.4.1 |

### CHANGELOG.md の書き方

```md
## [x.y.z] - yyyy-mm-dd

### Added   ← 機能追加
### Fixed   ← バグ修正
### Changed ← 仕様変更
### Removed ← 削除
```

## ビルド・デプロイ

変更後は必ず `npm run build` を実行し、`dist/todo-manager.html` を最新状態にしてからコミットすること。
